# PolyGraph v0.1.0 (alpha)

First public alpha. PolyGraph explores, audits, enforces, and compares software architecture across
**26 languages** — entirely locally.

## Highlights

- **Interactive graph** of modules/types/functions and their relationships (imports, calls,
  inheritance, instantiation, composition, dependency injection, JSX renders), GPU-rendered with a
  Vello (Rust→WASM, WebGPU) vector canvas.
- **Edge evidence & confidence** — click an edge to see where each relationship was observed
  (file · line · column), how many times, and the underlying symbol relationships.
- **Smart layout** — nested package/community containers, adaptive per-cluster layout, SCC
  collapse, semantic reduction (plus classic layered/tree/radial/circular/grid/force engines).
- **Impact analysis** — dependencies/dependents, shortest path, why-connected, and blast radius;
  a **Problems panel** of clickable architectural insights.
- **Architecture rules in CI** — `polygraph check` with `.polygraph.yml`, SARIF output, and
  `--baseline` for "fail only on new violations".
- **Graph diffing** — `polygraph diff` compares revisions / the working tree.
- **Query language**, **abstraction levels** (package/workspace from manifests), **exports**
  (DOT / GraphML / Mermaid / JSON / SVG / HTML report) + saved workspaces, and **editor
  integration** (inline source preview + open-at-line in VS Code / JetBrains, desktop only).

## Downloads

The desktop app is built per-architecture by the release workflow:

- **Windows x86_64** — installer (`.exe` / `.msi`) — also runs on Windows-on-ARM via emulation
- **macOS Apple Silicon** — `.dmg`
- **Linux x86_64** — `.AppImage` + `.tar.gz` (portable / Arch)
- **Linux aarch64** — `.AppImage`
- **`SHA256SUMS-*.txt`** — per-arch checksums for the bundles

> **Unsigned builds.** Bundles are unsigned and ship no auto-updater. Windows and macOS will show an
> "unknown publisher" / Gatekeeper warning — expected for unsigned alpha builds. See
> [docs/RELEASING.md](../RELEASING.md).

## Try it on

PolyGraph scans any local folder — point it at a project and paste the absolute path into
**Scan a folder**. Good first targets:

- **This repository** — a real multi-language project (TypeScript app + Rust `analyzer-core` and
  `vello-renderer` + tree-sitter language packs); a good way to see TS precision next to
  tree-sitter resolution.
- **Any monorepo** — switch to the **package**/**workspace** abstraction level to see module
  boundaries and cross-package dependencies.
- **A service with layering** — try `polygraph check` with a `.polygraph.yml` that forbids
  `client → server` imports or caps fan-in, and `polygraph diff` across two branches.

## Known limitations by language

PolyGraph uses two analysis strategies. **TypeScript/JavaScript** is type-resolved via the
TypeScript compiler; **every other language** uses declarative tree-sitter packs with **by-name
resolution** (a global unique-definer index, separator-agnostic). That difference drives what each
language captures:

| Language(s)                                                                                                    | Resolution                        | Notes                                                                                                                                                                                                             |
| -------------------------------------------------------------------------------------------------------------- | --------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| TypeScript / JavaScript (+ JSX, `.vue`, `.svelte`)                                                             | **Type-resolved**                 | Most precise: type-resolved calls, JSX `renders`, composition (`has`), DI (`injects`), framework/paradigm roles.                                                                                                  |
| Python                                                                                                         | Dotted/relative imports + by-name | Module resolution against the scanned set; calls/inheritance by name.                                                                                                                                             |
| Java, Kotlin, Rust, Go, Scala, C#, F#, C, C++, Objective-C, Swift, Zig, Haskell, Ruby, PHP, Dart, Julia, OCaml | **By-name** (tree-sitter)         | Defs + imports + calls/inheritance/instantiation by name. No type checker, so `has`/`injects`/JSX `renders` are TS/JS-only, and same-named symbols across modules can resolve **ambiguously** (surfaced as such). |
| Bash, Lua, R, Nix, SQL, WebAssembly text                                                                       | **By-name**, reduced edges        | Mostly definitions + imports/sourcing; fewer relationship types by nature.                                                                                                                                        |

General caveats for the tree-sitter path:

- **No type flow / generics** — composition and dependency-injection edges are TS/JS-only.
- **Dynamic code** — reflection, macros, metaprogramming, and runtime wiring are under-captured;
  references attribute to the innermost enclosing emitted symbol.
- **C/C++/Objective-C** — the preprocessor is not run; `#include` becomes a path-based import edge;
  templates are not type-resolved.
- **Rust** — manifest dependency discovery covers `[dependencies]`, `[dev-dependencies]`,
  `[build-dependencies]`, and `[target.*]`; macro-expanded relationships are not captured.

Adding precision to a language means graduating its pack to a code-backed provider — the plugin
boundary is designed for exactly that.

## Security

- **Runs entirely locally** — reads a folder from disk via a loopback-only sidecar; nothing is
  uploaded or persisted.
- **Tracked advisory (Linux desktop only):** `glib 0.18.5` —
  [RUSTSEC-2024-0429](https://rustsec.org/advisories/RUSTSEC-2024-0429). Linux build dependency
  only (not in the Windows/macOS apps, web, sidecar, or CLI); blocked upstream on Tauri's GTK4
  migration. See [SECURITY.md](../../SECURITY.md) and
  [#4](https://github.com/CapsaicinBunny/PolyGraph/issues/4).

## Notes

- The package is `"private": true`; the `polygraph` CLI runs from this repo and inside the desktop
  app. Registry publication (`bunx polygraph` / `npx polygraph`) would require removing `"private"`
  and publishing.
- Requires a WebGPU-capable browser (recent Chrome/Edge) for the in-browser graph canvas.
